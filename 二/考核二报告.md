# 必做二
## ollama
ollama部署的体验非常好，一行命令瞬间启动，能力测试和推理测试时逻辑清晰，幻觉和编造问题少，本地交互流畅度非常高
## ModelScope
modelscope部署的体验很差。首先部署方面就麻烦，我从魔搭社区下载了这个模型，配置好了环境，然后让ai帮我写部署代码。其次是使用体验，启动非常的慢，在IDE里运行代码启动要等差不多一分钟。
最后就是幻觉非常严重，容易胡言乱语。后面我手动降低temperature，然后在ai的指导下调节了max_new_tokens参数稳定模型和加了repetition_penalty=1.1重复惩罚，模型才算聪明了些
## 总结


| 维度         | Ollama                  | ModelScope (调参后)               |
|--------------|-------------------------|-----------------------------------|
| 部署复杂度   | 极低（一行命令）        | 中等（需写脚本）                  |
| 启动速度     | ≈2秒                    | ≈45秒                             |
| 内存占用     | 较小                  | 较大                            |
| 推理质量     | 高（低幻觉）            | 中高（调参后改善）                |
| 可定制性     | 低                      | 高（可改采样策略、量化等）        |
| 适用场景     | 个人本地聊天、快速测试  | 开发调试、集成到应用              |
## 原因
Ollama与ModelScope的体验差异，本质是应用场景与设计目标的根本不同Ollama定位为面向个人用户的极简本地推理工具，为实现开箱即用、流畅对话的核心目标，做了C++底层加速、上下文隔离、自动量化等极致优化，因此部署便捷、启动迅速、幻觉率低；而ModelScope是面向开发者的全流程模型开发平台，核心服务于模型下载、训练微调、企业私有化部署等工程化场景，保留了Python原生推理的完整灵活性与可扩展性，未针对本地对话做专项优化，因此在个人聊天场景下出现部署繁琐、启动缓慢、幻觉严重等问题，需手动调参才能改善，二者并无优劣之分，仅为不同场景的适配性选择。

# 心得体验
